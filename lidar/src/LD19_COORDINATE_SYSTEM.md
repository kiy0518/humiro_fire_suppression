# LD19 좌표계 및 각도 시스템

## 좌표계 정의

LD19는 **왼손 좌표계 (Left-Hand Rule)**를 따릅니다:

- **0도 위치**: 센서의 전면 (X축 방향)
- **원점**: 거리 측정 단위의 회전 중심
- **각도 증가 방향**: 시계 방향 (Clockwise)
- **각도 범위**: 0° ~ 360°

## 좌표계 다이어그램

```
        Y축 (90°)
         ↑
         |
         |
   180° ←┼→ 0° (X축, 전면)
         |
         |
        ↓
       270°
```

## 각도 계산

- **0°**: 센서 전면 (정면)
- **90°**: 왼쪽 (시계 방향)
- **180°**: 후면
- **270°**: 오른쪽

## 프로젝트 적용

### 현재 구현
- 각도 계산: `start_angle + i * 3.0f` (패킷당 12개 포인트, 3도 간격)
- 각도 정규화: 0° ~ 360° 범위로 정규화
- Front Distance: 0°에 가장 가까운 포인트 찾기

### 주의사항

1. **광학 윈도우**: 레이저 발사/수신을 위한 광학 윈도우가 필요하며, 외부 시스템에 의해 부분적으로 가려지면 거리 측정 성능에 영향을 줄 수 있습니다.

2. **0도 기준점**: 
   - LD19의 0도는 센서의 전면을 의미합니다
   - 드론에 설치할 때 센서의 전면이 드론의 전면과 일치하도록 설치해야 합니다
   - 설치 각도 오프셋이 있으면 보정이 필요합니다

3. **각도 범위**: 
   - LD19는 360° 전체 범위를 스캔합니다
   - 각도 범위가 제한되어 있다면 (예: 36° ~ 74°), 각도 계산 로직에 문제가 있을 수 있습니다

## 문제 해결

### 각도 범위가 제한된 경우
- 각도 계산 로직 확인
- 패킷 파싱 로직 확인
- 스캔 데이터 누적 로직 확인

### 0도 근처에 포인트가 없는 경우
1. 각도 오프셋 확인
2. LiDAR 설치 각도 확인
3. 각도 보정 추가
